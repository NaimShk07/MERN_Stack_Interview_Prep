# 2-Year Roadmap to Switch to FAANG/Product-Based Companies

**Profile:** MERN Developer, 1.5 years experience, 21 years old, Bachelor's in Commerce  
**Current Commitment:** 2 hours/day on weekdays, 4–5 hours/day on weekends  
**Goal:** Full-stack + backend mastery, DSA, System Design, DevOps, Cloud, Microservices, FAANG preparation

---

## Month 1-3: Strengthening Core DSA and JavaScript

- [✅] Learn/revise JavaScript fundamentals thoroughly (ES6+, async/await, closures, promises)
- [ ] Learn/revise Node.js and Express in depth
- [ ] Learn/revise MongoDB and Mongoose ORM
- [ ] Complete 1–2 DSA problems daily (arrays, strings, linked list)
- [ ] Start solving problems on LeetCode (Easy level)
- [ ] Build a small project applying Node.js + MongoDB

**Suggested Resources:**

- JS: You Don’t Know JS (book)
- DSA: “Grokking Algorithms” + LeetCode

---

## Month 4-6: Advanced DSA + System Design Basics

- [ ] Continue DSA (stack, queue, tree basics)
- [ ] Solve 3–4 LeetCode problems/day (easy → medium)
- [ ] Start understanding Big-O, time-space complexity
- [ ] Learn basics of system design: client-server, REST, APIs
- [ ] Read about caching, database indexing
- [ ] Build 1 full-stack project integrating APIs with DB

**Suggested Resources:**

- DSA: LeetCode, HackerRank
- System Design: “System Design Primer” (GitHub)

---

## Month 7-9: Backend Deep Dive + Advanced Projects

- [ ] Learn authentication & authorization (JWT, OAuth)
- [ ] Learn relational databases (PostgreSQL/MySQL) and ORM (Sequelize/TypeORM)
- [ ] Start Docker basics & containerization
- [ ] Learn microservices architecture concepts
- [ ] Solve 3–4 medium-level DSA problems/day
- [ ] Build a project with microservices + Docker

**Suggested Resources:**

- Docker Docs
- Microservices patterns: “Microservices Patterns” (book)

---

## Month 10-12: Cloud & DevOps Basics

- [ ] Learn AWS basics (EC2, S3, Lambda, RDS)
- [ ] CI/CD pipelines with GitHub Actions / Jenkins basics
- [ ] Kubernetes fundamentals (pods, deployments, services)
- [ ] Implement a project deploying microservices to AWS + CI/CD
- [ ] Solve 3–4 medium-level DSA problems/day

**Suggested Resources:**

- AWS Certified Cloud Practitioner course
- Kubernetes official docs

---

## Month 13-15: System Design + Advanced DSA

- [ ] Learn scalable system design (load balancer, caching, queues, database sharding)
- [ ] Practice high-level design for common apps (Twitter, Instagram, URL shortener)
- [ ] Solve 3–4 medium-hard DSA problems/day
- [ ] Participate in weekly mock interviews or coding contests

**Suggested Resources:**

- “Designing Data-Intensive Applications” (book)
- LeetCode Hard Problems

## Month 16-18: Full-Stack Mastery + Mock Interviews

- [ ] Build 2–3 real-world projects with React + Node + Microservices + Cloud
- [ ] Optimize code & architecture for scalability
- [ ] Solve 3–5 hard DSA problems/week
- [ ] Conduct 1–2 mock interviews/week

**Suggested Projects:**

- E-commerce site with microservices

- URL shortener + analytics dashboard

## Month 19-21: Focused FAANG Preparation

- [ ] Daily 2–3 LeetCode problems (medium + hard)
- [ ] Review system design patterns for interviews
- [ ] Optimize projects for performance & maintainability
- [ ] Continue mock interviews & peer coding sessions

**Suggested Resources:**

- “Cracking the Coding Interview”
- “Elements of Programming Interviews”

## Month 22-24: Final Touches & Applications

- [ ] Revise DSA & system design
